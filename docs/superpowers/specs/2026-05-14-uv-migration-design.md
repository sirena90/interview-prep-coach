# uv Migration Design

**Date:** 2026-05-14
**Branch:** uv-migration

## Goal

Replace `pip` + `requirements.txt` with `uv` as the sole package manager. `requirements.txt` is deleted; `pyproject.toml` + `uv.lock` become the source of truth.

## What Changes

| Before | After |
|---|---|
| `pip install -r requirements.txt` | `uv sync` |
| `streamlit run app.py` | `uv run streamlit run app.py` |
| `requirements.txt` | `pyproject.toml` + `uv.lock` |
| No lockfile | `uv.lock` (committed) |

## pyproject.toml Structure

```toml
[project]
name = "interview-prep-coach"
version = "0.1.0"
requires-python = ">=3.10"
dependencies = [
    "streamlit>=1.28.0",
    "anthropic>=0.39.0",
    "chromadb>=0.5.0",
    "pydantic>=2.0.0",
    "pypdf>=4.0.0",
    "python-dotenv>=1.0.0",
]

[dependency-groups]
dev = ["ruff"]
```

Install with `uv sync --group dev`. CI / contributors who only need to run the app use plain `uv sync`.

## Files Touched

- **Delete:** `requirements.txt`
- **Create:** `pyproject.toml`
- **Create:** `uv.lock` (generated by `uv sync`)
- **Update:** `CLAUDE.md` — setup/run commands
- **Update:** `README.md` — Quick start section
- **Update:** `ARCHITECTURE.md` — Running locally section

## Out of Scope

- Adding pytest, mypy or other dev tooling beyond ruff
- Changing the model or any application code
- Docker / CI changes
